# 医院管理系统就诊通知功能修改说明

## 一、功能概述

本次修改实现了以下功能：

1. **预约成功后立即发送所有后续提醒** - 当医生预约审核通过后，系统会立即创建并发送所有后续提醒通知
2. **用户接收状态判断** - 记录用户是否已读取通知
3. **通知发送失败的系统记录** - 详细记录每次通知的发送状态和失败原因
4. **重试机制** - 自动和手动重试发送失败的通知
5. **管理员后台查看和处理** - 提供完整的后台管理界面

## 四、通知类型

系统支持以下4种通知类型：

| 类型值 | 类型名称 | 发送时机 |
|--------|----------|----------|
| 1 | 预约成功通知 | 预约审核通过后立即发送 |
| 2 | 就诊前24小时提醒 | 就诊时间前24小时 |
| 3 | 就诊前1小时提醒 | 就诊时间前1小时 |
| 4 | 就诊前15分钟提醒 | 就诊时间前15分钟 |

## 五、发送状态

| 状态值 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 待发送 | 已创建但未到发送时间 |
| 1 | 发送中 | 正在发送 |
| 2 | 发送成功 | 通知已成功发送 |
| 3 | 发送失败 | 通知发送失败，可重试 |

## 六、核心功能说明

### 1. 预约成功后的通知创建逻辑

当医生预约审核通过时（YishengyuyueController.shBatch方法），系统会：

1. 立即创建并发送"预约成功通知"
2. 创建"就诊前24小时提醒"（如果就诊时间大于24小时）
3. 创建"就诊前1小时提醒"（如果就诊时间大于1小时）
4. 创建"就诊前15分钟提醒"（如果就诊时间大于15分钟）

### 2. 自动重试机制

NotificationServiceImpl中实现了两个定时任务：

- **retryFailedNotifications** - 每分钟检查一次发送失败的通知，自动重试（最多3次）
- **sendScheduledNotifications** - 每分钟检查一次待发送的通知，到达发送时间后自动发送

### 3. 后台管理功能

管理员可以在后台进行以下操作：

- 查看所有通知记录
- 按预约编号、接收账号、发送状态、读取状态筛选
- 查看通知详情
- 批量重试发送失败的通知
- 查看发送统计（总数、成功、失败、待发送、已读、未读）
- 删除通知记录

## 九、使用说明

### 1. 数据库初始化

执行 `server_code/sql/tongzhijilu_table.sql` 创建通知记录表

### 2. 后端部署

重新编译并部署后端代码：
```bash
cd server_code
mvn clean package
```

### 3. 前端部署

前端页面已创建，需要重新编译前端代码：
```bash
cd manage_code
npm run build
```

### 4. 菜单配置

在后台管理系统中添加"通知记录"菜单，路径为 `/views/tongzhijilu/list`

## 十、注意事项

1. 短信发送功能目前使用模拟实现（成功率90%），实际项目中需要接入短信服务商API
2. 定时任务每分钟执行一次，可根据实际需求调整频率
3. 最大重试次数为3次，可在NotificationServiceImpl中修改
