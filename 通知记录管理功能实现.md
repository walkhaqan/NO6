# 通知记录管理功能实现

## 功能概述
为管理员添加后台查看和处理通知记录的功能，包括查看通知记录列表、详情、重试发送通知、处理失败通知等功能。

## 实现内容

- 文件路径：`client_code/src/views/pages/tongzhijilu/list.vue`

  - 分页显示通知记录列表

## 使用说明

### 1. 查看通知记录列表
1. 管理员登录系统
2. 在左侧菜单中找到"通知记录管理"
3. 点击进入通知记录列表页面
4. 可以查看所有通知记录，包括通知编号、预约编号、医生账号、用户账号、通知类型、接收渠道、发送状态、处理状态等信息
5. 可以按条件筛选通知记录
6. 可以进行批量操作

### 2. 查看通知记录详情
1. 在通知记录列表页面，点击"详情"按钮
2. 进入通知记录详情页面
3. 可以查看通知记录的详细信息，包括失败原因、处理时间、处理人、备注信息等

### 3. 重试发送通知
1. 在通知记录列表页面，选择状态为"发送失败"的通知记录
2. 点击"重试发送"按钮
3. 系统会尝试重新发送通知
4. 如果重试次数超过3次，系统会自动将处理状态设置为"已放弃"

### 4. 处理失败通知
1. 在通知记录列表页面，选择状态为"发送失败"的通知记录
2. 点击"处理"按钮
3. 在弹出的对话框中，选择处理状态（已处理、已忽略、已放弃）
4. 输入备注信息
5. 点击"确定"按钮完成处理

### 5. 批量操作
1. 在通知记录列表页面，选择多条通知记录
2. 点击"批量重试"或"批量处理"按钮
3. 根据提示完成操作

## 注意事项

1. 通知记录会自动记录发送状态和重试次数
2. 重试发送次数最多为3次，超过3次后系统会自动将处理状态设置为"已放弃"
3. 处理失败通知时，需要填写处理状态和备注信息
4. 通知记录不可删除，只能标记为已处理或已忽略
5. 系统会自动根据预约信息创建通知记录
6. 通知记录的接收渠道可以是短信或邮件
7. 通知类型包括预约成功通知、就诊前一天提醒、就诊当天提醒等

## 扩展功能

1. 可以添加定时任务，定期处理发送失败的通知记录
2. 可以添加通知模板，自定义通知内容
3. 可以添加通知统计功能，统计发送成功率、处理率等指标
4. 可以添加通知日志，记录详细的发送过程
5. 可以添加通知黑名单，过滤无效的接收地址
